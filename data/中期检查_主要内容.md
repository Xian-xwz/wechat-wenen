## （二）主要研究内容

本毕业论文（设计）旨在设计并实现一个基于学习行为大数据的个性化学习辅导系统。该系统以解决在线教育中普遍存在的"一刀切"教学问题为核心目标，通过构建"数据采集-分析-决策-反馈"的智能闭环，实现因材施教。

鉴于系统涉及复杂的推荐算法验证与多维数据可视化展示，本研究采用"**Web端先行验证，小程序端适配发布**"的渐进式开发策略。目前主要的研究与设计内容包括以下五个方面：

### 1. 基于前后端分离的系统架构设计

为了确保系统的高扩展性与多端兼容性，本研究构建了基于 RESTful API 的前后端分离架构。

- **后端**：采用 Python 语言结合 Django REST Framework (DRF) 搭建服务器，负责业务逻辑处理、数据库交互及算法服务接口的封装。
- **前端**：目前阶段使用 Vue.js + Element Plus 构建 Web 端平台，利用 Web 端宽屏优势进行复杂功能的开发与调试；后续将基于 Uni-app 框架将核心模块迁移至微信小程序端，实现跨平台发布。

### 2. 多维学习行为数据的采集与前后端联动方案

本研究设计了一套完整的数据采集链路，解决"前端如何无感采集"与"后端如何高效接收"的技术难点。

#### 2.1 前端埋点技术实现

在 Web 端（Vue.js）开发了自定义指令（Directives），实现了对用户学习行为的"隐形埋点"。具体包括：

- **交互行为**：通过事件监听捕获鼠标点击、悬停（Hover）、滑动轨迹等数据。
- **专注度指标**：利用页面生命周期函数（onMounted, onBeforeUnmount）和 Page Visibility API 计算用户在知识点页面的有效停留时长与切换后台频率。

#### 2.2 前后端接口联动

设计了标准化的日志上报接口。前端将采集的行为数据封装为 JSON 格式（包含 user_id, event_type, timestamp, meta_data），通过 Axios 异步发送至 Django 后端的 API 接口。后端利用 Celery 异步任务队列处理日志入库，避免阻塞主业务线程，确保高并发下的系统稳定性。

### 3. 用户画像构建与混合推荐算法的实现步骤

本系统核心智能模块基于 Python 数据生态（Pandas、Scikit-learn）实现，具体分为画像建模与推荐计算两个阶段：

#### 3.1 用户画像建模

建立了动态更新的向量化标签体系。

- **静态标签**：基于注册信息（年级、教材版本）。
- **动态权重**：根据答题记录计算用户的"知识点掌握度矩阵"，并结合活跃时间段生成"学习习惯标签"。

#### 3.2 混合推荐算法流程

- **步骤一（解决冷启动）**：对于新用户，采用基于内容的推荐算法（Content-based）。系统计算用户选择的兴趣标签与题库资源标签（TF-IDF向量）的余弦相似度，推送Top-N资源。
- **步骤二（精准推荐）**：对于积累了一定数据的用户，切换为基于用户的协同过滤算法（User-based CF）。
  1. 构建"用户-题目"评分矩阵。
  2. 使用 Scikit-learn 计算用户间的皮尔逊相关系数（Pearson Correlation）。
  3. 基于相似用户的做题记录，预测当前用户对未做题目的潜在得分，并生成推荐列表。

### 4. 智能导学功能与自适应路径规划

为增强系统的交互性与指导性，研究并实现了基于规则引擎的智能导学功能。

#### 4.1 对话式交互反馈

在学习看板中植入了"智能助教"模块。系统不采用生硬的弹窗，而是通过对话框形式与用户交互（例如："检测到你刚才在'二叉树'知识点停留较久且答错，是否需要查看相关视频解析？"），引导用户确认学习需求，从而修正推荐策略。

#### 4.2 自适应路径生成（Adaptive Pathing）

- 构建了基于有向无环图（DAG）的知识图谱结构。
- **算法逻辑**：当监测到用户在"前置节点"（如：数组）的掌握度低于阈值（60%）时，算法自动锁定"后继节点"（如：链表）的进阶任务，并动态插入"补救路径"（基础概念回顾 -> 简单例题 -> 二次测试），形成个性化的学习闭环。

### 5. 学情可视化与自适应路径规划

研究如何将抽象的算法结果转化为用户可感知的界面元素。

- **数据可视化**：利用ECharts图表库，在Web端设计动态的学习看板，直观展示用户的知识图谱雷达图与学习趋势折线图，帮助用户清晰认知自身薄弱环节。
- **路径规划**：根据学情分析结果，由系统自动生成自适应的学习路径（如"先复习基础概念A，再练习进阶习题B"），并在前端界面以可视化的路线图形式进行引导。

### （二）毕业设计完成情况

#### 1. 学习概况数据大屏（核心亮点页面）

这是中期检查中最能体现工作量的页面，利用 Web 端的优势展示复杂的图表。对应开题报告中的"学习看板可视化"。

**功能描述**：直观展示用户的整体学习状态、知识点掌握情况及学习趋势。

**具体板块**：

- **核心指标卡片**：展示"累计学习时长"、"今日刷题数"、"平均正确率"、"连续打卡天数"。
- **知识图谱雷达图（ECharts）**：展示用户在不同知识点（如数据结构、算法、网络等）的掌握程度，直观反映薄弱环节。
- **学习趋势折线图**：展示近7日或近30日的学习活跃度与答题正确率变化曲线。
- **待办/推荐任务栏**：简要列出系统根据当前学情推荐的今日必做任务（如"复习二叉树"、"完成模拟测试"）。

#### 2. 在线答题与学习页面（数据采集核心）

这是系统产生数据的源头，也是实现"前端埋点"的关键页面。对应开题报告中的"学习行为数据采集"。

**功能描述**：提供题目练习或资源阅读环境，并在后台静默采集用户行为数据。

**具体板块**：

- **题目/内容展示区**：显示题干、选项或学习视频/文档。
- **交互操作区**：单选/多选按钮、提交答案按钮、上一题/下一题切换。
- **计时与状态栏**：显示当前题目花费时间、总进度条。
- **隐形埋点功能（技术重点）**：此板块用户不可见，但在代码层面需实现。
  - 记录鼠标悬停/页面停留时长（判断是否认真阅读）。
  - 记录选项修改轨迹（判断是否犹豫）。
  - 记录切换后台次数（判断是否专注）。

#### 3. 用户中心与冷启动页（基础模块）

对应开题报告中的"用户管理"及解决"冷启动问题"的策略。

**功能描述**：管理个人信息，并在注册初期收集用户偏好。

**具体板块**：

- **注册/登录页**：账号密码登录。
- **兴趣标签选择页（冷启动）**：新用户注册后弹窗，让用户选择感兴趣的领域或当前的知识水平（如"初学者"、"进阶"），用于初始推荐。
- **个人信息管理**：修改头像、昵称，查看积分或等级。

#### 4. 错题本与历史记录（辅助功能）

对应开题报告中的"错题本"功能。

**功能描述**：归纳整理用户的错误记录。

**具体板块**：

- **错题列表**：按时间或知识点分类展示历史错题。
- **错题详情弹窗**：点击列表项，展示原题、用户错误答案及正确解析。
